# AWS RDS 数据库初始化指南

## 📋 概述
本指南介绍如何使用 `aws_database_init.sql` 脚本在 AWS RDS MySQL 实例中初始化电影订票系统数据库。

## 📁 文件说明
- `aws_database_init.sql`: 完整的数据库初始化脚本，包含表结构、数据和关系约束

## 🚀 部署步骤

### 1. 准备工作
确保您已经创建了 AWS RDS MySQL 实例，并且可以连接到数据库。

### 2. 连接到 RDS 实例
使用 MySQL 客户端连接到您的 RDS 实例：
```bash
mysql -h <your-rds-endpoint> -P 3306 -u <admin-username> -p
```

### 3. 执行初始化脚本
在 MySQL 客户端中执行脚本：
```sql
SOURCE aws_database_init.sql;
```

或者使用命令行：
```bash
mysql -h <your-rds-endpoint> -P 3306 -u <admin-username> -p < aws_database_init.sql
```

### 4. 验证安装
脚本执行完成后，会自动显示验证信息，包括：
- 表创建状态
- 数据计数
- 外键关系验证

## 📊 数据库内容

### 用户表 (users)
- 5个用户记录（1个管理员，4个普通用户）
- 密码已使用 BCrypt 加密
- 包含用户名、邮箱、电话等信息

### 电影表 (movies)
- 5部电影记录
- 包含中文标题和描述
- 评分、时长、导演等信息

### 放映时间表 (showtimes)
- 14个放映场次
- 不同影厅和价格
- 座位信息和可用性

### 预约表 (bookings)
- 8个预约记录
- 包含座位号、状态、价格等
- 与用户和放映时间关联

### 电影类型表 (movie_genres)
- 目前为空表
- 为未来扩展预留

## 🔧 技术特性

### 字符编码
- 使用 UTF-8 (utf8mb4) 字符集
- 支持中文字符正确显示
- 兼容 AWS RDS MySQL 8.0+

### 数据完整性
- 完整的外键约束
- 唯一键约束（用户名、邮箱）
- 自动递增主键

### 安全考虑
- 密码使用 BCrypt 哈希
- 敏感信息已脱敏处理

## ⚠️ 注意事项

1. **备份现有数据**: 在生产环境中执行前请备份现有数据
2. **权限要求**: 需要数据库管理员权限
3. **字符编码**: 确保客户端和服务器都使用 UTF-8 编码
4. **网络连接**: 确保 RDS 安全组允许您的 IP 访问

## 🔍 故障排除

### 连接问题
- 检查 RDS 端点和端口
- 验证安全组设置
- 确认用户名和密码

### 字符编码问题
- 确保客户端使用 UTF-8
- 检查 RDS 参数组中的字符集设置

### 执行错误
- 查看 MySQL 错误日志
- 检查磁盘空间和内存使用情况

## 📞 支持

如果遇到问题，请检查：
1. AWS RDS 控制台的状态
2. MySQL 错误日志
3. 网络连接和安全组配置
